🎹 Music Trainer
Интерактивный тренажер для обучения игре на музыкальных инструментах (пианино и гитара) с визуализацией в реальном времени и виртуальным помощником.

💡 Описание идеи и ценности

● Проблема: Обучение музыке часто кажется скучным и перегруженным теорией. Начинающим игрокам сложно сопоставить ноты на бумаге с реальными клавишами или струнами.

● Решение: Проект предлагает геймифицированный подход который мотивирует игрока.

🛠 Стек технологий

● Движок: Cocos Creator 3.8.8 (используется для рендеринга 2D/3D графики и управления сценами).

● Язык программирования: TypeScript

● Дополнительные библиотеки: pitchy (снятие звука)

● Графика и UI:

  ○ Figma: создание интерфейса и векторных ассетов (иконки, кнопки).
  
  ○ Standard Shaders: для настройки материалов 3D-пианино.
  
● Хостинг: GitHub Pages.

🏗 Компоненты системы и взаимодействие

Система построена на модульной архитектуре внутри Cocos Creator:

1. Frontend (Клиентская часть):

○ Scene Manager: управляет переходами между главным экраном и режимами инструментов.

○ Mascot Controller: автономный модуль, отвечающий за Idle-анимации (покачивание), логику приветствия и выдачу уникальных подсказок без повторений.

○ Input System: обрабатывает клики пользователя по UI-элементам и 3D-объектам через встроенную систему событий Cocos.

2. Взаимодействие:

○ Взаимодействие компонентов происходит через Event System и прямые ссылки в скриптах (@property).

○ Для плавности интерфейса используются компоненты UIOpacity и система Tween, обеспечивающие "рассеивание" диалоговых облаков и динамику персонажа.

🚀 Как запустить проект

Локальный запуск (Разработка):

1. Установите Cocos Creator 3.8.8.

2. Склонируйте репозиторий

3. Установите библиотеку "pitchy"

4. Откройте папку проекта через Cocos Dashboard.

5. Нажмите кнопку Play в верхней части редактора для запуска в браузере.

Сборка (Build):

1. Перейдите в меню Project -> Build.

2. Выберите платформу Web Mobile или Web Desktop.

3. Установите галочку MD5 Cache (обязательно для корректного обновления файлов в сети).

4. Нажмите Build, затем запакуйте содержимое папки build/ в ZIP-архив.
